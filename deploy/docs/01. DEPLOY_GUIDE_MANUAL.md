# 🚀 Deploy Script - Portal Service

Script deploy cho Umbraco CMS Portal với .NET 10.0 và PostgreSQL.

## 📋 Yêu Cầu

- Docker & Docker Compose
- SSH access đến remote server (cho deployment)
- Build context: Solution root (`srcs/`)

## 🔧 Cấu Hình

### Environment File

Copy và chỉnh sửa `.env.portal`:

```bash
cd scripts/envs/dev
cp .env.portal.example .env.portal
# hoặc tạo .env.portal.local để override
nano .env.portal
```

**Các biến quan trọng**:

| Biến | Mô Tả | Ví Dụ |
|------|-------|-------|
| `IMAGE_NAME` | Tên Docker image | `dmst-portal` |
| `IMAGE_TAG` | Tag version | `v0.0.1` |
| `BUILD_CONFIG` | .NET build config | `Release`, `Debug` |
| `APP_VERSION` | Application version | `1.0.0` |
| `LOCAL_COMPOSE_FILE` | Compose file name | `portal.yml` |
| `SERVERS` | Remote servers | `user@host` |

## 🚀 Sử Dụng

### Build Only Mode

Build và save Docker image, **KHÔNG deploy**:

```bash
./deploy.sh portal dev --build-only
```

Với custom build config:

```bash
./deploy.sh portal dev --build-only --build-config Debug --app-version 1.2.0
```

### Pre-built Mode (Build trên máy Dev)

Build bằng `dotnet publish` trên host rồi COPY vào Docker image (không cần multi-stage build):

```bash
# Build + deploy
./deploy.sh portal dev v0.2.0 --prebuilt

# Chỉ build image, không deploy
./deploy.sh portal dev v0.2.0 --prebuilt --build-only

# Với custom config
./deploy.sh portal dev v0.2.0 --prebuilt --build-config Debug --app-version 1.2.0
```

> **Yêu cầu**: .NET 10.0 SDK phải được cài trên máy dev.
> **Flow**: `dotnet publish` → `docker build` (Dockerfile.portal.prebuilt) → save → deploy

### Deploy to Development

```bash
# Deploy với tag mặc định từ .env
./deploy.sh portal dev

# Deploy với custom tag
./deploy.sh portal dev v0.2.0

# Deploy với cleanup old images
./deploy.sh portal dev --cleanup
```

### Deploy to Production

```bash
# Deploy với tag cụ thể
./deploy.sh portal prod v1.0.0

# Deploy với custom build settings
./deploy.sh portal prod v1.0.0 --build-config Release --app-version 1.0.0 --cleanup
```

## 🔄 Build Process

Script thực hiện các bước sau:

1. **Load Configuration**: Đọc `.env.portal` hoặc `.env.portal.local`
2. **Validate Prerequisites**: Kiểm tra SSH connection (nếu deploy)
3. **Build Image**: 
   - **Default**: Build với Docker Compose (multi-stage Dockerfile)
   - **Pre-built** (`--prebuilt`): `dotnet publish` trên host → `docker build` với Dockerfile.portal.prebuilt
   - Pass `BUILDCONFIG` và `VERSION` build args
   - Skip nếu image đã tồn tại
4. **Save Image**: Export image thành `.tar` file
5. **Deploy** (nếu không phải build-only):
   - SCP image file lên server
   - Load image trên server
   - Update compose file với image tag
   - Down old container
   - Up new container
   - Health check
6. **Cleanup**: Xóa old images (nếu enabled)

## 📝 Build Arguments

Script map environment variables sang Dockerfile build args:

| Env Variable | Build Arg | Default | Description |
|--------------|-----------|---------|-------------|
| `BUILD_CONFIG` | `BUILDCONFIG` | `Release` | .NET build configuration |
| `APP_VERSION` | `VERSION` | `1.0.0` | Application version |

**Dockerfile hỗ trợ**:

```dockerfile
ARG BUILDCONFIG=Release
ARG VERSION=1.0.0

RUN dotnet build --configuration ${BUILDCONFIG} /p:Version=${VERSION}
```

## 🔐 SSH Authentication

### Key-based (Recommended)

```bash
# Trong .env.portal
SSH_AUTH_METHOD="key"
SSH_KEY_PATH="${HOME}/.ssh/dmst_key.pem"
```

### Interactive (Manual)

```bash
# Trong .env.portal
SSH_AUTH_METHOD="interactive"
# SSH_KEY_PATH không cần thiết
```

Script sẽ nhắc nhập password cho mỗi SSH/SCP command.

## 🏥 Health Check

Enable health check để verify deployment:

```bash
# Trong .env.portal
HEALTH_CHECK_ENABLED="true"
HEALTH_CHECK_PORT="8080"
HEALTH_CHECK_PATH="/umbraco/api/health"
HEALTH_CHECK_TIMEOUT="60"
```

Nếu health check fail, deployment vẫn được xem là thành công (warning only).

## 📦 Image Management

### Local Images

Saved tại: `${LOCAL_IMAGE_SAVE_DIR}/${IMAGE_NAME}_${IMAGE_TAG}.tar`

```bash
# Ví dụ
~/Desktop/docker/images_backup/dmst-portal_v0.0.1.tar
```

### Cleanup Old Images

```bash
# Tự động cleanup khi deploy (nếu --cleanup flag)
IMAGE_RETENTION_HOURS="168"  # 7 days

# Hoặc manual
docker image prune -af --filter "until=168h"
```

## 🐛 Troubleshooting

### Build lỗi "context not found"

```bash
# Kiểm tra LOCAL_COMPOSE_DIR trong .env.portal
LOCAL_COMPOSE_DIR="/d/Workspace/GTSC2026/26.dmst.c12.portal/srcs/compose"

# Directory phải tồn tại và có Dockerfile.portal
ls -la $LOCAL_COMPOSE_DIR
```

### Image không build lại

Script skip build nếu image tag đã tồn tại:

```bash
# Option 1: Xóa local image
docker rmi dmst-portal:v0.0.1

# Option 2: Dùng tag mới
./deploy.sh portal dev v0.0.2

# Option 3: Force rebuild bằng cách xóa tag trong compose file
```

### SSH connection failed

```bash
# Test SSH manually
ssh -i ~/.ssh/dmst_key.pem user@host

# Check prerequisites
./check_prerequisites.sh portal dev
```

### Health check timeout

```bash
# Tăng timeout
HEALTH_CHECK_TIMEOUT="120"

# Hoặc disable
HEALTH_CHECK_ENABLED="false"

# Kiểm tra endpoint manual
curl http://SERVER_IP:8080/umbraco/api/health
```

## 💡 Best Practices

1. **Versioning**: Luôn dùng semantic versioning (`v1.0.0`)
2. **Testing**: Build local trước với `--build-only`
3. **Backup**: Backup database trước khi deploy production
4. **Rollback**: Giữ previous image tags để rollback nhanh
5. **Secrets**: Không commit `.env.portal.local` vào git
6. **Logs**: Review deploy logs sau mỗi deployment
7. **Health Check**: Enable cho production deployments

## 📚 Ví Dụ Workflows

### Development Workflow

```bash
# 1. Build và test local
./deploy.sh portal dev --build-only --build-config Debug

# 2. Test image local
docker run -it --rm dmst-portal:v0.0.1

# 3. Deploy to dev server
./deploy.sh portal dev
```

### Production Workflow

```bash
# 1. Build release version
./deploy.sh portal prod v1.0.0 --build-only --build-config Release --app-version 1.0.0

# 2. Deploy với cleanup
./deploy.sh portal prod v1.0.0 --cleanup

# 3. Verify deployment
curl http://PROD_SERVER:8080/umbraco
```
