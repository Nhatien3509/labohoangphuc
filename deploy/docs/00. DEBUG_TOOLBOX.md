# 🔍 Debug Toolbox

Container chứa các công cụ debug để kiểm tra kết nối network, database, Redis từ **bên trong Docker network**.

## Khi nào dùng?

- Container không kết nối được database/Redis nhưng không rõ nguyên nhân
- Cần kiểm tra DNS resolution giữa các container
- Cần verify port mở/đóng từ góc nhìn container
- Debug network policy, firewall rules trong Docker network
- Kiểm tra HTTP endpoint từ internal network

## Quick Start

```bash
cd deploy/templates

# Build toolbox image
docker compose -f debug.yml build

# Chạy check-all (kiểm tra tất cả kết nối từ env vars trong debug.yml)
docker compose -f debug.yml run --rm toolbox check-all

# Interactive shell (đầy đủ tools)
docker compose -f debug.yml run --rm toolbox bash
```

## Commands

| Command | Mô tả | Ví dụ |
|---------|--------|-------|
| `check-all` | Kiểm tra tất cả từ env vars | `toolbox check-all` |
| `pg` | Test PostgreSQL | `toolbox pg db-host 5432 user mydb` |
| `redis` | Test Redis | `toolbox redis redis-host 6379` |
| `http` | Test HTTP endpoint | `toolbox http http://app:8080/health` |
| `port` | Test TCP port | `toolbox port hostname 443` |
| `dns` | DNS lookup | `toolbox dns container-name` |
| `bash` | Interactive shell | `toolbox bash` |

> **Lưu ý**: `toolbox` = `docker compose -f debug.yml run --rm toolbox`

## Environment Variables

Cấu hình trong `debug.yml` hoặc truyền qua `-e`:

| Variable | Default | Mô tả |
|----------|---------|--------|
| `PG_HOST` | `portal-dev-sqlserver` | PostgreSQL hostname |
| `PG_PORT` | `1433` | PostgreSQL port |
| `PG_USER` | `sa` | PostgreSQL user |
| `PG_DB` | `H04.Portal.DB` | Database name |
| `PGPASSWORD` | — | PostgreSQL password |
| `REDIS_HOST` | `be-dev-redis` | Redis hostname |
| `REDIS_PORT` | `6379` | Redis port |
| `REDIS_PASSWORD` | — | Redis password |

| `CHECK_HOSTS` | `portal-dev-minio:9000` | host:port pairs (space-separated) |

## Các kịch bản debug thường gặp

### 1. Container không kết nối được database

```bash
# Bước 1: Kiểm tra DNS — container name có resolve được không?
docker compose -f debug.yml run --rm toolbox dns portal-dev-sqlserver

# Bước 2: Kiểm tra port — database có listen không?
docker compose -f debug.yml run --rm toolbox port portal-dev-sqlserver 1433

# Bước 3: Kiểm tra auth — credentials đúng không?
docker compose -f debug.yml run --rm -e PGPASSWORD=mypassword toolbox pg portal-dev-sqlserver 5432 sa H04.Portal.DB
```

### 2. Redis connection timeout

```bash
# Check port
docker compose -f debug.yml run --rm toolbox port be-dev-redis 6379

# Check Redis auth
docker compose -f debug.yml run --rm toolbox redis be-dev-redis 6379 Abcd@123456
```

### 3. Health check fail

```bash
# Test HTTP endpoint từ trong network
docker compose -f debug.yml run --rm toolbox http http://umbraco-portal-app:8080/umbraco/api/health

# Hoặc curl chi tiết
docker compose -f debug.yml run --rm toolbox bash
curl -v http://umbraco-portal-app:8080/umbraco/api/health
```

### 4. Kiểm tra kết nối đến service bên ngoài

```bash
docker compose -f debug.yml run --rm toolbox bash

# DNS
dig google.com
nslookup external-api.example.com

# Port
ncat -zv external-api.example.com 443

# TLS certificate
openssl s_client -connect external-api.example.com:443 -servername external-api.example.com </dev/null

# Traceroute
tracepath external-api.example.com
```

### 5. Packet capture (advanced)

```bash
# Chạy với --cap-add NET_ADMIN
docker compose -f debug.yml run --rm --cap-add NET_ADMIN toolbox bash

# Capture traffic
tcpdump -i eth0 -n host portal-dev-sqlserver
tcpdump -i eth0 -n port 6379
tcpdump -i eth0 -n -A host portal-dev-minio  # xem HTTP payload
```

### 6. Network scan

```bash
docker compose -f debug.yml run --rm toolbox bash

# Scan subnet
nmap -sn 172.18.0.0/16

# Scan ports trên host
nmap -p 1-1000 portal-dev-sqlserver
```

## Công cụ có sẵn

| Tool | Mục đích |
|------|----------|
| `curl`, `wget` | HTTP/HTTPS requests |
| `dig`, `nslookup`, `host` | DNS queries |
| `ping`, `tracepath` | ICMP connectivity |
| `telnet` | Interactive TCP connection |
| `nmap`, `ncat` | Port scanning, netcat |
| `tcpdump` | Packet capture (cần `--cap-add NET_ADMIN`) |
| `openssl` | TLS/SSL testing |
| `jq` | JSON parsing |
| `psql` | PostgreSQL client |
| `redis-cli` | Redis client |

## Kết nối vào network khác

Mặc định toolbox join `umbraco-portal-network`. Để debug network khác, sửa `debug.yml`:

```yaml
networks:
  portal-network:
    external: true
    name: <tên-network-cần-debug>  # đổi tên network ở đây
```

Hoặc chạy trực tiếp không qua compose:

```bash
docker run --rm -it --network <network-name> debug-toolbox bash
```
